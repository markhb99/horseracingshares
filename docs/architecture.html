Key Data Flows

Buyer Enquiry

1Buyer fills in enquiry form on horse detail page
2Vercel API validates input (Zod), checks AFSL gate
3Rate-limit check: max 3 per email per horse per 24 h
4Enquiry row inserted into Supabase (dedup trigger fires)
5Consent ledger entries written
6After response: forward email to syndicator via Resend
7After response: buyer confirmation email via Resend
8Lead score recomputed and upserted to Loops

Syndicator Lists a Horse

1Syndicator completes listing form (name, PDS, share tiers)
2Vercel API creates Stripe checkout session
3Syndicator pays listing fee via Stripe
4Stripe fires payment.succeeded webhook to Vercel
5Horse row created in Supabase (status: pending_review)
6Admin reviews and approves → status: active
7DB trigger enqueues upsert into search_outbox
8Indexer worker pushes horse document to Typesense

Buyer Searches for a Horse

1Buyer visits /browse (URL params carry filters + sort)
2Next.js server component reads URL params
3Server calls Typesense with filter_by + facet_by
4Typesense returns hits + facet counts in < 50 ms
5Page renders server-side (SEO-friendly, no loading flash)
6Buyer adjusts filters → URL updates → new server render

Saved Search Digest (Cron)

1Vercel cron fires daily 07:00 AEST / weekly Monday 08:00
2API reads saved_search rows due for this cadence
3Checks marketing_email consent per user via Supabase RPC
4Typesense searched for horses newer than last_sent_at
5Deduplicates matches across multiple saved searches per user
6Resend sends one digest email per user (batched)
7last_sent_at advanced; saved_search_run row written

Service Connection Reference

Browser → External

  • Vercel — all page loads (HTTPS + CDN)
  • Supabase Auth — sign in, sign up, OAuth
  • PostHog — client-side event tracking
  • Mux — horse promo video streaming
  • Stripe.js — embedded payment UI

Vercel Server → Services

  • Supabase DB — all data reads and writes
  • Supabase Auth — session verification (JWT)
  • Supabase Storage — signed image/PDF URLs
  • Typesense — horse search queries (SSR + cron)
  • Resend — enquiry, confirmation, digest emails
  • Loops — lead profile upserts + campaigns
  • Stripe — checkout creation + webhook verify

Automated / Background

  • Stripe fires webhook to Vercel on payment
  • Supabase DB triggers write to search_outbox
  • Indexer worker polls outbox → pushes to Typesense
  • Vercel crons fire search + email digest daily/weekly
  • GitHub Actions deploys to Vercel on push to main
